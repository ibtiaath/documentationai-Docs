title: Top up with Wayl (Visa/Mastercard)
description: Add wallet balance using the Wayl payment gateway from a card.
---

Wayl is the EBTE3ATH payment gateway for Iraq. Users top up their wallet by entering their card details on a hosted Wayl checkout page; on success, the balance is credited automatically.

<Callout type="note">
  Commission: **2.5% + 600 IQD** for local cards, **3.5% + 600 IQD** for international cards. These are added on top of the deposit amount and shown to the user before they confirm.
</Callout>

## The flow

<Mermaid>
sequenceDiagram
    Client->>API: POST /payments/wayl/deposit/
    API-->>Client: payment_link_url + reference_id
    Client->>Wayl: Redirect to checkout
    Wayl-->>Client: 3DS challenge → success
    Wayl->>API: Webhook (signed)
    API-->>API: Credit user's wallet
    Client->>API: GET /payments/wayl/status/{reference_id}/
    API-->>Client: 200, status: completed
</Mermaid>

## Steps

<Steps>
  <Step title="Get the deposit configuration (commission preview)">
    ```http
    GET /api/v1/payments/wayl/deposit-config/
    ```

    Response:
    ```json
    {
      "min_amount": "5000.00",
      "max_amount": "10000000.00",
      "commission": {
        "local": { "percent": 2.5, "fixed": 600 },
        "international": { "percent": 3.5, "fixed": 600 }
      }
    }
    ```

    Show the user the total they'll be charged including commission before continuing.
  </Step>

  <Step title="Create a payment link">
    ```http
    POST /api/v1/payments/wayl/deposit/
    Authorization: Bearer <access-token>
    ```

    <ParamField path="amount" type="string" required>
      Amount in IQD as a decimal string. Excludes commission.
    </ParamField>

    <ParamField path="card_type" type="string" required>
      `local` or `international`. Drives which commission applies.
    </ParamField>

    Response:
    ```json
    {
      "reference_id": "WAYL-2026-000789",
      "payment_link_url": "https://checkout.wayl.iq/p/abc123...",
      "amount": "25000.00",
      "commission": "1225.00",
      "total_charged": "26225.00",
      "status": "pending",
      "expires_at": "2026-04-28T11:00:00Z"
    }
    ```
  </Step>

  <Step title="Open the checkout">
    Redirect the user (web) or open a WebView (mobile) to `payment_link_url`.

    Wayl returns to a redirect URL with the result. The mobile app intercepts:

    ```
    https://api.ibtiaath.com/api/v1/payments/wayl/redirect/?reference_id=WAYL-...&status=success
    ```
  </Step>

  <Step title="Confirm via polling">
    The webhook is the source of truth — the redirect can be missed. Poll the status endpoint until it resolves.

    ```http
    GET /api/v1/payments/wayl/status/WAYL-2026-000789/
    ```

    States:

    | Status | Meaning |
    |---|---|
    | `pending` | Awaiting user action on the checkout |
    | `processing` | Wayl is settling the transaction |
    | `completed` | Wallet has been credited |
    | `failed` | Transaction declined; nothing was charged |
    | `expired` | Link expired (15 minutes) |
    | `refunded` | Admin issued a refund |
  </Step>
</Steps>

## Polling implementation

Poll every 2 seconds for up to 90 seconds. Stop on any terminal state.

<Tabs>
  <Tab title="JavaScript">
    ```ts
    async function waitForPayment(reference: string) {
      const terminals = ['completed', 'failed', 'expired'];
      for (let i = 0; i < 45; i++) {
        const { data } = await api.get(`/payments/wayl/status/${reference}/`);
        if (terminals.includes(data.status)) return data;
        await new Promise((r) => setTimeout(r, 2000));
      }
      throw new Error('Payment timed out');
    }
    ```
  </Tab>
  <Tab title="Dart">
    ```dart
    Future<Map<String, dynamic>> waitForPayment(String reference) async {
      const terminals = ['completed', 'failed', 'expired'];
      for (var i = 0; i < 45; i++) {
        final r = await http.get(
          Uri.parse('$base/payments/wayl/status/$reference/'),
          headers: {'Authorization': 'Bearer $access'},
        );
        final data = jsonDecode(r.body);
        if (terminals.contains(data['status'])) return data;
        await Future.delayed(const Duration(seconds: 2));
      }
      throw Exception('Payment timed out');
    }
    ```
  </Tab>
</Tabs>

## Pay-for-X variants

The same Wayl flow can pay directly for a specific resource (skipping the wallet step):

```http
POST /api/v1/payments/wayl/flight/      # pay for a flight booking
POST /api/v1/payments/wayl/program/     # pay an application fee
POST /api/v1/payments/wayl/scholarship/ # pay a scholarship fee
```

Each takes the resource ID and creates a payment link bound to that resource. On webhook completion, the resource is marked paid directly.

## Webhook security

EBTE3ATH receives Wayl webhooks at `/api/v1/payments/webhooks/wayl/`:

- **Per-payment-link secrets** — each link has its own HMAC secret (not a global one).
- **Replay protection** — the `WebhookEvent` model dedupes by event ID.
- **Atomic credit** — the wallet update happens inside `select_for_update()` so duplicate webhooks can't double-credit.

You don't need to handle the webhook yourself; the platform does. Just poll the status endpoint after the user returns from checkout.

## Refunds (admin)

Admins can refund a completed payment from the dashboard or via:

```http
POST /api/v1/payments/wayl/admin/refund/
```

Body: `{ "reference_id": "WAYL-...", "reason": "..." }`. The amount is debited from the user's wallet (if topped up) and a refund is initiated with Wayl. Track via:

```http
GET /api/v1/payments/wayl/admin/refund-status/{refund_id}/
```
